# 异构时序预测模型融合预训练架构 (PackerMoEFusion) 技术方案

## 一、 背景与动机
在复杂时空序列预测任务中，单一架构模型（如基于注意力的 Transformer 或基于分解的 MLP）受限于特定的归纳偏置（Inductive Bias），难以兼顾长程依赖建模与局部趋势捕捉。引入异构模型融合架构，通过大规模数据驱动构建“元表征中心”，自动协同、解耦并重组不同基模型的知识，是提升预测精度与鲁棒性的必然趋势。

---

## 二、 核心技术挑战：从基础融合到大规模预训练

实现高效异构融合及预训练面临以下三大维度的科学瓶颈：

### 1. 语义隔离与表征流形对齐 (Alignment)
*   **挑战描述**：异构模型的隐藏层输出呈现显著的语义异质性，分布在完全不同的流形空间。
*   **核心难题**：
    *   **维度失配**：不同模型在不同抽象层级生成的特征维度不一。
    *   **语义漂移**：在预训练中，如何识别不同架构中关于“周期”、“趋势”等物理概念的共性表征，同时抑制不同模型对同一特征定义不同产生的漂移？
*   **后果**：直接线性加权或拼接会导致严重的梯度干扰，损失各模型的特有归纳偏置。

### 2. 特征冗余与计算开销的非线性增长 (Scaling)
*   **挑战描述**：在多通道场景下，特征 Token 数量随模型数 $M$ 与通道数 $C$ 几何级增长。
*   **核心难题**：
    *   **计算爆炸**：全量特征融合的复杂度达 $O((M \times C)^2)$，在超多通道任务中算力开销远超基模型。
    *   **信息增益难题**：异构模型输出中包含大量共性（冗余）信息及预测噪声，如何在海量特征中实现“去粗取精”？
*   **后果**：融合层可能成为系统瓶颈，且由于低信噪比导致预测性能劣化。

### 3. 动态非平稳性下的专家冲突与特化 (Routing)
*   **挑战描述**：时间序列具有显著的非平稳性，不同模型在不同时段（如波动段 vs 周期段）性能表现各异。
*   **核心难题**：
    *   **动态路由决策**：如何构建自适应机制，实时调度最匹配的“专家”逻辑，避免静态权重的“优劣相抵”。
    *   **专家塌陷 (Expert Collapse)**：在大规模预训练中，系统易倾向于通过少数“万能特征”处理所有任务，导致神经单元失去功能特化。
    *   **冻结约束下的冲突**：在保留基模型原始知识（Frozen）的前提下，如何化解不同模型对同一预测窗口产生的互斥信号？

---

## 三、 技术方案：基于 PackerMoEFusion 的融合架构

针对上述挑战，本项目提出 PackerMoEFusion 架构，核心通过“投影对齐-结构压缩-动态路由”实现高效融合。

### 1. 异构特征语义对齐方案 (Projector-based Alignment)
*   **多头线性投影器 (Multi-head Projectors)**：为每个冻结的基模型设计独立投影头，将异构特征映射至统一的 $D_{fusion}$ 语义空间。
*   **对比对齐辅助损失 (Contrastive Alignment Loss)**：引入自监督对比学习，强制不同模型对同一输入序列生成的特征在流形空间中保持语义一致性，降低融合层的理解成本。

### 2. 基于 Latent Queries 的结构化信息压缩 (TokenPacker)
*   **潜在查询向量压缩 (Latent Query Compression)**：摒弃 Self-Attention，采用 Cross-Attention 机制。通过一组可学习的潜在查询向量（Queries），从 $M \times C$ 个异构 Token 中主动吸取关键预测因子。
*   **复杂度优化**：将计算复杂度从 $O((M \times C)^2)$ 降低至 $O(Q \times M \times C)$（$Q$ 为固定查询数），实现处理超多维、超长序列任务的线性扩展能力。

### 3. 场景感知与专家特化的动态路由 (Soft MoE Fusion)
*   **全微分 Soft MoE 路由**：引入 Soft MoE 机制，通过可微分的权重矩阵实现 Token 级的“细粒度信用分配”，解决非平稳数据下的动态权重难题。
*   **功能分化诱导**：设计基于信息熵的负载平衡损失 (Load Balancing Loss)，强制诱导不同专家解耦出特定模式（如高频突发 vs 长程趋势），化解专家塌陷。

### 4. 两阶段预训练与一致性保障策略
*   **第一阶段（对齐期）**：冻结基模型与 MoE 专家，仅训练 Projectors 与 TokenPacker，确立基础的跨模型语义映射。
*   **第二阶段（激活期）**：释放 MoE 路由参数，在跨域数据上进行端到端优化，诱导专家特化。
*   **分布漂移补偿 (Dynamic RevIN)**：在输出端集成基于历史统计量的 RevIN 还原模块，确保在面对量级迥异的跨域数据时，输出始终符合物理逻辑。

### 5. 参数高效的下游适配 (PEFT)
*   **路由自适应微调 (Router-only Tuning)**：下游微调时仅更新 MoE 路由权重，以极小参数代价快速识别当前任务的最优基模型组合。
*   **结构弹性适配 (Elastic Query Resizing)**：支持根据任务通道数动态调整 `num_queries`，实现对变长、变维度任务的零重训适配。
